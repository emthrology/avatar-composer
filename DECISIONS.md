# DECISIONS — 의사결정 로그 (ADR)

avatar-composer 분기 과정에서 내린 결정과 근거. "왜 이렇게 했지?"가 나올 때 참조.
배경 검증은 drei-avatar-project `poc/asset-swap` 브랜치(`b99d156`).

---

## ADR-001 — 에셋 스왑은 별도 프로젝트로 분리
**날짜:** 2026-06-16 · **상태:** 채택

**맥락.** drei-avatar-project는 "임의 VRM 1개를 통째 로드해 셰이더/색/톤/조명을 만지고 VTuber로
말하게" 하는 프로젝트. 자산이 monolithic(VRoid가 Face/Body/Hair를 merged export)이라는 전제 위에 섬.
여기에 얼굴 조형/헤어/옷 **에셋 스왑**을 요구받음.

**결정.** 기반(R3F/drei/VRM 로딩/조명/store)만 포크하고 **별도 프로젝트로 시작.**

**근거.** 에셋 스왑은 정반대 전제를 요구 — 통짜 1개가 아니라 **고정 베이스 + 모듈 파츠 라이브러리**,
모델별 스켈레톤이 아니라 **공유 스켈레톤**, 그리고 "임의 사용자 업로드" **포기**. 같은 레포에 넣으면
두 전제가 충돌(개발 원칙①비퇴행 위반). 기능 추가가 아니라 **전제의 교체**라 분리가 맞음.

## ADR-002 — 헤어/옷 스왑은 임의 업로드 모델과 양립 불가
**날짜:** 2026-06-16 · **상태:** 채택 (제약 확정)

**맥락.** "헤어=장착이니 가볍지 않나"는 반론 검증. male_sample.vrm 파싱 결과 헤어는 별도 메시/노드 +
스프링본 33개로 묶여 있음.

**결정.** 에셋 스왑은 **우리가 authored한 고정 베이스 + 파츠 라이브러리에서만** 가능. 임의 업로드 모델엔
지원하지 않음.

**근거.** 임의 모델엔 (a) 새 헤어 지오메트리를 합성할 수 없고, (b) 크로스모델 graft는 head 본 크기·
스프링본이 달라 실패. "헤어=장착"은 반만 맞음 — 얼굴 지오메트리 변형은 아니지만, 장착이 가벼운 것도
아님(에셋 사전 제작 + 스키닝/스프링본 통합 필요). path A(통짜 VRM 여러 개)·C(런타임 graft) 기각.

## ADR-003 — 에셋 전략 = 고정 베이스 직접 제작
**날짜:** 2026-06-16 · **상태:** 채택

**결정.** VRoid 파라메트릭 활용·외부 SDK(Avaturn 등) 대신 **베이스 1구 + 파츠 라이브러리를 직접 authored.**

**근거.** VRoid 파라메트릭은 "프리셋 선택"이지 파츠 단위 조합이 아님. 외부 SDK는 RPM 종료(2026.01
Netflix)·Avaturn 유료. 직접 제작이 통제권 최대 + 무료. **웹 앱은 쉬운 30%, 진짜 본체는 오프라인 에셋
저작 파이프라인**임을 인지(이전 avatar-pipeline의 vroid_to_glb.py 같은 변환이 본체가 되는 구조).

## ADR-004 — 베이스는 인하우스(VRoid), 외주 라인아이템에서 제외
**날짜:** 2026-06-16 · **상태:** 채택

**맥락.** 베이스도 3D 자산이라 "외주 작업 아닌가?"라는 의문.

**결정.** 베이스(마네킨)는 **VRoid Studio로 인하우스 제작.** 외주는 **모듈 파츠(헤어·옷)만.**

**근거.** 베이스는 본 이름·바인드 포즈·스케일·네이밍을 **"정의"**하는 컨벤션 키스톤 — 만드는 주체가
**컨벤션 락 소유권**을 가짐. 외주에 넘기면 컨벤션이 외부 손에 가고 베이스가 틀어지면 모든 파츠가 연쇄로
틀어짐(최상위 리스크). VRoid는 무료 GUI(3D 아티스트 불필요)고 VRM humanoid + `J_Bip_*` + A-pose를
**표준으로 그대로 채택** → 사양서가 "우리 표준 따르라"가 아니라 "이 베이스에 맞춰라"로 단순화(외주 리스크↓).
VRoid는 모듈 파츠는 못 만듦(통짜 bake) → 파츠만 Blender 외주가 맞음.

## ADR-005 — 베이스-퍼스트 순서
**날짜:** 2026-06-16 · **상태:** 채택

**결정.** 원래 "(a)설계문서 → 에셋"을 뒤집어 **베이스 생산 → 문서 → 외주 파츠 → 완전검증** 순.

**근거.** 베이스가 문서를 구체화. 문서를 먼저 쓰면 본 이름·포즈를 **추정**으로 적지만, 베이스를 먼저
만들면 **실측값**으로 못박음. (실제로 male_base.vrm 파싱으로 ASSET_SPEC을 실측 확정함)

## ADR-006 — 얼굴 조형은 Blender 셰이프키 별도, 후순위
**날짜:** 2026-06-16 · **상태:** 채택

**결정.** 얼굴 조형(비율) 모프는 베이스 머리에 **Blender 셰이프키로 별도 저작.** 헤어·옷보다 후순위.

**근거.** VRoid 얼굴 슬라이더는 export 시 메시에 **baked** → 모프로 안 나옴(male_base 파싱: `Fcl_*`
표정 모프 57종만, 조형 모프 0). 실시간 웹에서 자유 sculpt는 불가. → 고정 베이스에 명명 셰이프키 슬라이더.
