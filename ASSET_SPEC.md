# ASSET_SPEC — 에셋 저작 사양 (외주 핸드오프) v1.0

avatar-composer는 **베이스(캐릭터) + 모듈 파츠 조립** 방식이다. 외주 아티스트는 우리가 제공하는
**해당 캐릭터 베이스** 위에 헤어·옷을 모델링·스키닝한 뒤 **파츠만** 내보낸다.
아래 값은 베이스에서 **실측한 고정 규약**이다. 어긴 파츠는 런타임 조립이 깨진다.

> **다중 베이스.** 규약(§1)은 베이스 불가지 — 남자1(`male_base.vrm`)·여자1(`female1/female_base.vrm`)이
> **동일 규약**(VRM1.0·54본·`J_Bip_*`·A-pose·57모프·MToon)을 공유한다(여자1 검증 완료). 파츠는 베이스에
> 종속하므로(남자1 셔츠 ≠ 여자1 셔츠) 그 캐릭터 베이스 위에서 저작한다. 신장만 베이스별로 다를 수 있다.

## 0. 작업 전제
- 무에서 만들지 않는다. **제공 베이스 위에서** 제작한다. (게임 의상 에셋 표준 워크플로)
- 검증 게이트: 납품 파츠를 런타임 조립(`bind(baseSkeleton)`)에 끼워 **팔/머리 움직임에서 자연스럽게 변형되면 합격**.

## 1. 스켈레톤 / 좌표 — 절대 변경 금지 (컨벤션 락, male_base.vrm 실측)
| 항목 | 고정값 |
|---|---|
| 포맷 | **VRM 1.0** (`VRMC_vrm` spec 1.0) |
| 스켈레톤 | **humanoid 54본**, VRoid `J_Bip_*` 네이밍 |
| 바인드 포즈 | **A-pose** (VRoid 표준) |
| 단위/스케일 | 미터, 신장 **1.756m**, scale **1.0** (transform apply 필수) |
| 원점/정면 | 발밑 바닥 중심 / 정면 **+Z** |
| 머티리얼 | **MToon** |

- 본 추가·삭제·리네임·계층 변경 **금지**. 파츠는 베이스의 동일 본에 웨이트.
- 주요 본 이름: `J_Bip_C_Hips/Spine/Chest/UpperChest/Neck/Head`,
  `J_Bip_L|R_Shoulder/UpperArm/LowerArm/Hand`, `J_Bip_L|R_UpperLeg/LowerLeg/Foot/ToeBase`,
  손가락 `J_Bip_L|R_(Thumb|Index|Middle|Ring|Little)1~3`. (전체 맵은 베이스 파일 기준)

## 2. 파츠 공통 규칙
- **1 파츠 = 1 스킨드 메시**, 베이스 동일 본에 스키닝.
- **본 영향 ≤ 4개/버텍스** (glTF 표준). 초과 웨이트 정리·정규화.
- **노멀 필수**, **삼각화(triangulate)**, n-gon 금지.
- **머티리얼 = MToon**, lit/shade 컬러 + base color 텍스처. 외곽선 폭 베이스와 통일.
- **UV**: 파츠별 자체 0~1 UV + 자체 아틀라스, **2의 거듭제곱**(1024/2048), 1파츠 1머티리얼 권장.
- **네이밍 프리픽스**: 헤어 `Hair_*`, 상의 `Tops_*`, 하의 `Bottoms_*`, 신발 `Shoes_*`.
- **폴리 예산**: 헤어 ≤ 15k tri, 의상 1점 ≤ 20k tri (협의 가능).

## 3. 의상 전용 — 살 비침(poke-through) 대책
런타임이 옷 밑 바디를 숨겨야 겹쳐 보이지 않는다. 둘 중 **하나** 제공:
- **(권장) 숨김 영역** — 옷이 덮는 바디 버텍스를 버텍스 그룹 **`Hide_Body`** 로 마킹해 함께 전달.
- **(대안) 완전 봉합** — 어떤 포즈에서도 안 비치게 완전히 덮어 모델링.
- 바인드 + 차렷/팔들기에서 **관통 없을 것** (허용 ~2mm).

## 4. 헤어 전용 — 부착 방식
- **고정 헤어**: head 본 단일 웨이트 → 리지드 부착.
- **흔들리는 헤어**: **VRM 스프링본 체인**으로 제작 + 콜라이더 포함, **VRM으로 내보내** 스프링 설정이
  메타데이터(`VRMC_springBone`)에 실리게. (런타임이 읽어 베이스에 병합) — 조인트 네이밍 별도 합의.

## 5. 얼굴 조형 — 모프 타깃 (베이스 머리에 한함, 후순위)
- ⚠️ VRoid 얼굴 슬라이더는 export 시 메시에 **baked** → 조형 모프로 안 나온다. **Blender 셰이프키로 별도 저작.**
- 베이스 머리 메시에 **명명된 블렌드셰이프**, 중립=0, 슬라이더 0~1 보간.
- 표정 모프 `Fcl_*`(57종 기존)와 **이름 충돌 금지**. 예: `Shape_FaceWidth`, `Shape_JawLength` … (목록 확정 필요)

## 6. 내보내기 / 자가검수 체크리스트
- 포맷: 의상·고정헤어 = **GLB** / 스프링 헤어 = **VRM**. Y-up, +Z, 모디파이어 apply, 텍스처 임베드, **Draco 끄기**.
- 납품 합격 조건:
  - [ ] 본 이름 베이스와 **완전 일치** (추가/삭제/리네임 0)
  - [ ] 바인드 포즈 = A-pose
  - [ ] 스케일 1.0 / transform apply
  - [ ] 버텍스당 본 영향 ≤ 4
  - [ ] 노멀 정상 / 삼각화 / n-gon 0
  - [ ] 의상: `Hide_Body` 그룹 or 완전 봉합 + 관통 검사 통과
  - [ ] 프리픽스 네이밍 준수
  - [ ] 1파츠 1파일 + 텍스처 임베드
